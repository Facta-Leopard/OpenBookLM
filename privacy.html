Privacy Policy

Last updated: 2026-05-05

This policy explains how OpenBookLM handles local Vault documents, document translation, chat content, provider sign-in, local credentials, and iPad advertising.

English

Local data

  • OpenBookLM processes user-selected Vault documents locally for text extraction, indexing, retrieval, source opening, text/Markdown editing where supported, citations, reference review, and document translation.
  • Supported extraction formats include .txt, .md, .markdown, .pdf, .rtf, .docx, .epub, and .hwpx. Rich formats are opened as read-only extracted text previews, so layout, images, and complex tables may not be preserved.
  • Document translation creates a new app-managed translated file and does not overwrite the original. Generated translations may be stored locally and can be renamed or exported by the user.
  • The app may store settings, chat sessions, session scope preferences, vault connection data, security-scoped bookmarks, note metadata, translation metadata, and vault index metadata on your device.
  • OpenBookLM does not operate a developer-hosted sync server for your notes, indexes, generated translations, or chat sessions.

Provider modes

  • When Vault Search is enabled, relevant local note context may be included in the prompt sent to the selected generation provider.
  • Apple On-Device / Foundation Models mode handles prompts, responses, and document translation segments on device through Apple's on-device model path.
  • ChatGPT Browser Sign-In, OpenAI-compatible API, and other cloud-backed modes may transmit prompts, chat context, and selected note context to that provider. Local API mode sends prompts and context to the endpoint you configure.
  • Third-party providers process submitted data under their own privacy policies and terms.

Credentials, ads, retention

  • Credentials, API keys, and provider tokens are stored in the system Keychain where applicable and can be replaced or removed from provider settings or by clearing local data.
  • The iPad version may display Google AdMob banner ads. Google Mobile Ads SDK may process data such as approximate location derived from IP address, device identifiers, diagnostics, performance data, advertising data, and ad interaction data under Google's policies. The macOS version does not include AdMob.
  • Where required, the iPad version requests ad consent through Google's User Messaging Platform and provides Privacy & Compliance settings.
  • Local app data remains on your device until you delete it through app controls or remove the app. Support emails are retained only as needed to respond, keep support history, or meet applicable obligations.
  • The developer does not sell personal data. For deletion or consent-withdrawal requests involving developer-managed support records, contact support from the email address used for the original request.

日本語

ローカルデータ

  • OpenBookLM は、ユーザーが選択した Vault 文書を、テキスト抽出、インデックス作成、検索、原文表示、対応形式でのテキスト/Markdown 編集、引用、参照確認、文書翻訳のために端末内で処理します。
  • 対応形式は .txt, .md, .markdown, .pdf, .rtf, .docx, .epub, .hwpx です。リッチ形式は読み取り専用の抽出テキストプレビューとして開かれ、レイアウト、画像、複雑な表は保持されない場合があります。
  • 文書翻訳は元文書を上書きせず、アプリ管理の新しい翻訳ファイルを作成します。生成された翻訳はローカルに保存され、名前変更または書き出しできます。
  • アプリは、設定、チャットセッション、セッション範囲設定、Vault 接続情報、セキュリティスコープ付きブックマーク、ノートメタデータ、翻訳メタデータ、Vault インデックスメタデータを端末に保存する場合があります。
  • OpenBookLM は、ノート、インデックス、翻訳、チャットセッションを同期する開発者運営サーバーを提供していません。

Provider モード

  • Vault Search が有効な場合、関連するローカルノートのコンテキストが選択中の Provider へ送るプロンプトに含まれることがあります。
  • Apple On-Device / Foundation Models は、プロンプト、応答、文書翻訳セグメントを Apple のオンデバイスモデル経路で処理します。
  • ChatGPT Browser Sign-In、OpenAI 互換 API、その他クラウド Provider では、プロンプト、チャットコンテキスト、選択ノートコンテキストが Provider に送信される場合があります。Local API は設定したエンドポイントへ送信します。
  • 第三者 Provider は送信データを各社のプライバシーポリシーと規約に従って処理します。

認証情報、広告、保持

  • 認証情報、API キー、Provider トークンは可能な場合はシステム Keychain に保存され、Provider 設定またはローカルデータ削除で置換・削除できます。
  • iPad 版では Google AdMob バナー広告が表示される場合があります。Google Mobile Ads SDK は Google のポリシーに従い、IP に基づくおおよその位置、端末識別子、診断、パフォーマンスデータ、広告データ、広告操作データを処理する場合があります。macOS 版には AdMob は含まれません。
  • 必要な場合、iPad 版は Google User Messaging Platform で広告同意を求め、Privacy & Compliance 設定を提供します。
  • ローカルアプリデータは、ユーザーがアプリ内機能で削除するかアプリを削除するまで端末に保持されます。サポートメールは問い合わせ対応、履歴維持、適用義務に必要な範囲でのみ保持されます。
  • 開発者は個人データを販売しません。開発者管理のサポート記録に関する削除または同意撤回は、元の問い合わせアドレスから連絡してください。

Español

Datos locales

  • OpenBookLM procesa en el dispositivo los documentos de Vault seleccionados para extracción de texto, indexación, búsqueda, apertura de fuentes, edición de texto/Markdown cuando corresponda, citas, revisión de referencias y traducción.
  • Los formatos compatibles son .txt, .md, .markdown, .pdf, .rtf, .docx, .epub y .hwpx. Los formatos enriquecidos se abren como vistas de texto extraído de solo lectura, por lo que diseño, imágenes y tablas complejas pueden no conservarse.
  • La traducción crea un archivo traducido nuevo gestionado por la app y no sobrescribe el original. Puede guardarse localmente, renombrarse o exportarse.
  • La app puede guardar ajustes, sesiones de chat, preferencias de sesión, datos de conexión de Vault, marcadores con permisos, metadatos de notas, traducciones e índice en el dispositivo.
  • OpenBookLM no opera un servidor de sincronización del desarrollador para notas, índices, traducciones o chats.

Modos de Provider

  • Con Vault Search activado, el contexto local relevante puede incluirse en el prompt enviado al Provider seleccionado.
  • Apple On-Device / Foundation Models procesa prompts, respuestas y segmentos de traducción en el dispositivo mediante la ruta de modelo de Apple.
  • ChatGPT Browser Sign-In, API compatible con OpenAI y otros modos en la nube pueden enviar prompts, contexto de chat y notas seleccionadas al Provider. Local API envía datos al endpoint configurado.
  • Los Providers de terceros procesan los datos enviados según sus propias políticas y términos.

Credenciales, anuncios y conservación

  • Credenciales, claves API y tokens se guardan en Keychain cuando corresponde y pueden reemplazarse o eliminarse desde ajustes o borrando datos locales.
  • La versión iPad puede mostrar banners Google AdMob. Google Mobile Ads SDK puede procesar ubicación aproximada por IP, identificadores, diagnósticos, rendimiento, datos publicitarios e interacción con anuncios según las políticas de Google. La versión macOS no incluye AdMob.
  • Cuando sea necesario, iPad solicita consentimiento mediante Google User Messaging Platform y ofrece ajustes Privacy & Compliance.
  • Los datos locales permanecen hasta que el usuario los borra o elimina la app. Los correos de soporte se conservan solo para responder, mantener historial o cumplir obligaciones aplicables.
  • El desarrollador no vende datos personales. Para eliminación o retirada de consentimiento sobre registros de soporte gestionados por el desarrollador, escriba desde el correo original.

  • Die Übersetzung erstellt eine neue appverwaltete Datei und überschreibt das Original nicht. Sie kann lokal gespeichert, umbenannt oder exportiert werden.
  • Die App kann Einstellungen, Chatsitzungen, Sitzungspräferenzen, Vault-Verbindungsdaten, Security-Scoped Bookmarks, Notizmetadaten, Übersetzungsmetadaten und Indexmetadaten auf dem Gerät speichern.
  • OpenBookLM betreibt keinen entwicklerseitigen Sync-Server für Notizen, Indizes, Übersetzungen oder Chats.

Provider-Modi

  • Bei aktivem Vault Search kann relevanter lokaler Kontext in den Prompt an den ausgewählten Provider aufgenommen werden.
  • Apple On-Device / Foundation Models verarbeitet Prompts, Antworten und Übersetzungssegmente auf dem Gerät über Apples On-Device-Modellpfad.
  • ChatGPT Browser Sign-In, OpenAI-kompatible API und andere Cloud-Modi können Prompts, Chatkontext und ausgewählten Notizkontext an den Provider senden. Local API sendet Daten an den konfigurierten Endpoint.
  • Drittanbieter verarbeiten übermittelte Daten nach ihren eigenen Datenschutzrichtlinien und Bedingungen.

Zugangsdaten, Werbung, Aufbewahrung

  • Wo erforderlich, fragt iPad über Google User Messaging Platform nach Werbeeinwilligung und bietet Privacy & Compliance Einstellungen.
  • Lokale Daten bleiben bis zur Löschung durch den Nutzer oder App-Entfernung auf dem Gerät. Support-E-Mails werden nur für Antwort, Supporthistorie oder Pflichten aufbewahrt.
